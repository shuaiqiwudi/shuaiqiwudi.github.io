一、 计算机数据类型总览

在大多数编程语言（如 C, C++, Java, Python 等）中，数据类型主要分为以下几大类：

  1. 整数类型 (Integer): 用于存储没有小数部分的数字。
    • byte, short, int, long：区别主要在于占用的内存大小和能表示的数值范围。
  2. 字符类型 (Character): 用于存储单个字符。
    • char：例如 ‘A’, ‘b’, ‘1’。
  3. 布尔类型 (Boolean): 用于逻辑判断。
    • bool：只有两个值，true (真) 或 false (假)。
  4. 浮点类型 (Floating Point): 用于存储带小数的数字。
    • float (单精度)
    • double (双精度)

二、 什么是“浮点数”？ (Floating Point)

简单来说，浮点数就是计算机用来表示实数（即带小数点的数）的一种方式。

1. 核心概念：科学计数法

要理解浮点数，首先要理解科学计数法。

比如，数字 123.45 可以写成：

1.2345×102

在计算机内部，由于它是二进制的，所以浮点数是基于二进制的科学计数法来存储的：

±尾数×2指数

2. 存储结构 (IEEE 754 标准)

计算机并不会直接存 “12.34” 这个形状，而是将有限的比特位（0和1）分成三部分来存：

  1. 符号位 (Sign, S): 决定是正数还是负数（0为正，1为负）。
  2. 指数位 (Exponent, E): 决定小数点“浮动”的位置（即数值的大小范围）。
  3. 尾数位 (Mantissa/Fraction, M): 决定数值的精度（即数字具体是多少）。

三、 单精度 (Float) vs 双精度 (Double)

这两个术语的区别主要在于占用的内存位数不同，从而导致了精度和数值范围的巨大差异。

1. 单精度浮点数 (float)

  • 字面意思： “Single Precision”，即使用标准的 32 位（4字节）来存储一个数。
  • 结构分配：
    • 1位符号
    • 8位指数
    • 23位尾数
  • 有效数字： 大约 6-7 位 十进制数字。这意味着，如果你存 1.23456789，最后几位可能就不准了。
  • 适用场景： 对精度要求不高，但对内存极其敏感的场景（如简单的图形处理、大规模低精度科学计算）。

2. 双精度浮点数 (double)

  • 字面意思： “Double Precision”，即使用双倍的内存，64 位（8字节）来存储一个数。
  • 结构分配：
    • 1位符号
    • 11位指数
    • 52位尾数
  • 有效数字： 大约 15-16 位 十进制数字。它的精度非常高。
  • 适用场景： 绝大多数现代编程场景。默认情况下，代码中的小数（如 3.14）都会被当作 double 处理。

四、 核心对比表：一目了然

为了方便记忆，我为你总结了一个对比表：

特性 Float (单精度) Double (双精度)
占用内存 32位 (4字节) 64位 (8字节)
有效数字(精度) 约 7 位 约 15-16 位
数值范围 ≈10−38 到 1038 ≈10−308 到 10308
处理速度 在旧硬件上较快，现代CPU上差距极小 现代计算机的标准处理方式
主要用途 节省内存、GPU图形渲染 科学计算、日常业务逻辑

五、 为什么叫“浮”点数？

这是一个非常形象的名字。

  • 定点数 (Fixed Point): 想象一下，如果我们规定小数点必须固定在第3位，那么 123.45 和 0.12 的表示方式就很死板。
  • 浮点数 (Floating Point): 小数点的位置是可以**移动（浮动）**的。

通过调整指数，小数点可以在尾数之间左右浮动。

  • 指数变大，小数点向右浮（数变大）。

  • 指数变小，小数点向左浮（数变小）。

    这使得计算机可以用固定的位数，表示出极大（如银河系的距离）或极小（如原子的半径）的数值。

六、 避坑指南：精度丢失问题

这是新手最容易踩的坑。浮点数并不总是精确的。

经典案例：

在编程中，0.1 + 0.2 往往不等于 0.3。

这是因为 0.1 在二进制中是一个无限循环小数（就像十进制里的 1/3 = 0.3333…），计算机存不下无限的位数，只能截断。

给你的建议：

  1. 涉及金钱计算时（如银行系统、电商价格）： 千万不要用 float 或 double！ 请使用专门的十进制类型（如 Java 的 BigDecimal 或 Python 的 Decimal）。
  2. 一般科学计算/游戏开发： 使用 double 通常足够安全。
  3. 比较相等时： 不要写 if (a == b)，而要写 if (abs(a - b) < 0.000001)（判断两个数是否足够接近）。

我们将通过一个经典的例子：把十进制的 12.5 转换成计算机里的二进制浮点数（IEEE 754 标准）。

这个过程分为三步走：

  1. 整数部分转换（除以2取余）
  2. 小数部分转换（乘以2取整）
  3. 拼凑与标准化（移位并填入内存）

第一步：整数部分转换 (12)

规则：除以 2，取余数，直到商为 0，然后倒序排列。

我们要转换 12：

被除数 除数 商 余数
12 2 6 0
6 2 3 0
3 2 1 1
1 2 0 1

结果： 从下往上读，即 1100。

第二步：小数部分转换 (0.5)

规则：乘以 2，取出整数部分，剩下的小数继续乘，直到小数为 0（或达到精度限制），然后顺序排列。

我们要转换 0.5：

  1. 0.5×2=1.0
    • 取出整数：1
    • 剩下小数： 0.0 (结束！)

结果： 从上往下读，即 .1。

合并结果：

十进制的 12.5 = 二进制的 1100.1

第三步：标准化 (科学计数法)

就像十进制我们要把小数点移到第一个数字后面一样，二进制也要移。

我们需要把 1100.1 的小数点向左移动 3 位，变成 1.xxxxx 的形式：

1.1001×23

这里有两个关键信息：

  1. 指数 (Exponent): 是 3 (因为向左移了3位)。
  2. 尾数 (Mantissa): 是 1001 (小数点后面的部分)。

第四步：装入内存 (IEEE 754 格式)

现在我们要把这些信息填入 float (32位) 的三个格子里：

1. 符号位 (1位)

  • 因为 12.5 是正数，所以填 0。

2. 指数位 (8位) —— 这里有个小机关！

计算机为了能同时表示正指数和负指数，制定了一个偏移量 (Bias)。对于 32 位浮点数，偏移量是 127。

  • 我们的真实指数是 3。
  • 存储指数 = 真实指数 + 偏移量 = 3+127=130 。
  • 把 130 转换成二进制：10000010。
  • 所以填入 10000010。

3. 尾数位 (23位) —— 还有一个小机关！

二进制科学计数法的第一位永远是 1 (因为它是 1.xxxxx )。既然永远是 1，聪明的计算机科学家决定省略不存，这样可以省下一位来存更多精度。

  • 我们只存小数点后面的部分：1001。
  • 后面不够的位数全部补 0，补满 23 位。
  • 所以填入 10010000000000000000000。

最终结果

把上面三部分拼起来，这就是 12.5 在计算机内存里真正的样子：

0 10000010 10010000000000000000000

(符号) (指数) (尾数)

⚠️ 为什么会出现 0.1 + 0.2 != 0.3？

学会了“乘以 2 取整”法，你就明白这个千古难题了。

让我们试着转换 0.1：

  1. 0.1×2=0.2 -> 取 0
  2. 0.2×2=0.4 -> 取 0
  3. 0.4×2=0.8 -> 取 0
  4. 0.8×2=1.6 -> 取 1 (剩 0.6)
  5. 0.6×2=1.2 -> 取 1 (剩 0.2) <-- 也就是在这里，变回了 0.2！
  6. 0.2×2=0.4 -> 取 0
  7. … 无限循环

0.1 在二进制里是一个无限循环小数： 0.00011001100110011...

但是计算机的 float 只有 23 位尾数，double 只有 52 位。存不下无限的位数，只能咔嚓一下截断。

这一截断，精度就丢了。所以计算机里的 0.1 其实大约是 0.10000000149…。

总结

  1. 整数用“除以2”。
  2. 小数用“乘以2”。
  3. 有些小数（如 0.1）在二进制里是无限循环的，这是浮点数不精确的根源。
