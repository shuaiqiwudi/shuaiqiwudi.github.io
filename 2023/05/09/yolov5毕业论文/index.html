将队员分为两个队伍的思路

  1. 获取正常的数据集 labels文件存放不分开的球员位置
  2. 获取labels文件的坐标位置
  3. 将坐标位置转化为截图能识别的像素位置
  4. 截取所有目标队员 保存到文件夹中
  5. 根据文件夹中的截取的图品内容 识别出截取的球员颜色
  6. 根据颜色进行分类 如果是白色 则在该txt文件中获取对应的坐标
  7. 修改这个坐标的分类为对应颜色球员
  8. 将更改过的labels内容进行train训练

获取球员的上半部分

但是有很多背景的像素影响识别球员衣服的颜色，我们可以将图象分为两个cluster，其中一个代表background_cluster,另一个代表player_shirt_cluster,我们就可以平均各自cluster的颜色

导入KMeans 进行分簇

但是仍然不知道background的cluster代表的编号是什么，但是通常背景都在四个角落的位置，可以获取他们的cluster分类的id序号

由此就可以获得对应的player_cluster_id,background_cluster_id,于是就可以使用kmeans获取对应cluster的质心 即球员的T恤颜色

实现

  1. 首先根据第一帧的所有球员颜色，再使用Kmean++进行颜色分类，将player的球衣颜色分为两个颜色，存储到team_colors[]

  2. 写一个识别每个球员颜色的方法，即上述的内容

  3. 识别每个球员，使用kmeans.predict方法预测球员属于的队伍，并记录在player_teams_dict[]中，方便下次直接读取

  4. 最后在main方法中，将识别出的team_id和team_color添加到tracks中

4. Camera Movement Estimator

灰度模式即使用单一色调来表示图像，与位图模式不同，不像位图只有0和1，使用256级的灰度来表示图像，一个像素相当于占用8为一个字节，每个像素值使用0到255的亮度值代表，其中0为黑色，255为白色，相当于从黑->灰->白的过度，通常我们所说的黑白照片就是这种模式，与位图模式相比，能表现出一定的细节，占用空间也比位图模式较大
